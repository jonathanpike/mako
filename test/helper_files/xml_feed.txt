        <title>Migrating from Boostrap-Sass v3.3.5 to Neat 2.0.0</title>
        <description>&lt;p&gt;A long time ago, I opened an &lt;a href=&quot;https://github.com/jonathanpike/jonathanpike.github.io/issues/2&quot;&gt;issue&lt;/a&gt; that simply said:&lt;/p&gt;

&lt;blockquote&gt;
  &lt;p&gt;SCSS features aren’t being used effectively. Needs refactoring.&lt;/p&gt;
&lt;/blockquote&gt;

&lt;p&gt;When I first started making my theme, I decided to use &lt;a href=&quot;http://getbootstrap.com/&quot;&gt;Bootstrap&lt;/a&gt; to speed up development. Bootstrap is a wonderful framework, making websites look great right out of the box.  Since it’s so full featured, it also included a lot of elements that I would never use for this site.  Why not explore some of Sass’ interesting features and remove what I didn’t need.&lt;/p&gt;

&lt;p&gt;I decided to do the obvious thing and search &lt;em&gt;“lightweight grid framework”&lt;/em&gt; on Google, and I found &lt;a href=&quot;http://neat.bourbon.io/&quot;&gt;Neat&lt;/a&gt; by Thoughtbot.  Neat uses Sass mixins instead of concrete classes, like Bootstrap.  This would be a great opportunity to learn more about Sass’ mixin system.  This also turned out to be the biggest learning curve.&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;The Container&lt;/strong&gt;&lt;/p&gt;

Bootstrap defines it through a class, either &lt;code class=&quot;highlighter-rouge&quot;&gt;.container&lt;/code&gt; for fixed-width grids or &lt;code class=&quot;highlighter-rouge&quot;&gt;.container-fluid&lt;/code&gt; for full-width grids. All of the options for the container are taken care of for you with either of those classes.  With Neat, I had to define my own container class (called &lt;code class=&quot;highlighter-rouge&quot;&gt;.container&lt;/code&gt;, which I carried over from Bootstrap) and then mix in Neats &lt;code class=&quot;highlighter-rouge&quot;&gt;grid-container&lt;/code&gt;:&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;The Columns&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;Bootstrap has several different classes for columns.  I was using &lt;code class=&quot;highlighter-rouge&quot;&gt;col-xs-[x]&lt;/code&gt; and &lt;code class=&quot;highlighter-rouge&quot;&gt;col-md-[x]&lt;/code&gt; to define columns for desktop and mobile.  My main content container (the one you’re reading in right now), for example, used &lt;code class=&quot;highlighter-rouge&quot;&gt;col-xs-12 col-md-8&lt;/code&gt; to make it span 8 of 12 columns on desktop and 12 of 12 columns on mobile.&lt;/p&gt;

&lt;p&gt;Neat, out of the box, doesn’t provide you with different sized columns defined by breakpoints for mobile, desktop, etc.  Instead, it provides you the tools to make those decisions for yourself.  This was the hardest part, for me, when I switched from Bootstrap to Neat.&lt;/p&gt;

At first, I thought that I would use the default &lt;code class=&quot;highlighter-rouge&quot;&gt;$neat-grid&lt;/code&gt; for desktop and define a custom &lt;code class=&quot;highlighter-rouge&quot;&gt;$mobile-neat-grid&lt;/code&gt; for my mobile needs.  That ended up not working as I expected, although I realized later that my mistake was the order my media queries were in: media queries need to be ordered from smallest to biggest, but I was ordering from biggest to smallest. What I ended up doing was exactly the opposite.  I defined a &lt;code class=&quot;highlighter-rouge&quot;&gt;$large-neat-grid&lt;/code&gt; as follows:&lt;/p&gt;

Easy and clear.  The transition was made even easier because I had already defined classes on all of my main container elements, so I just had to add the Neat mixins to each of these to replicate the layout I had made with Bootstrap.&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;Other Styles&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;One thing I didn’t realize was how much Bootstrap defined things like font styles and spacing within divs.  When I removed it, the whole layout looked really weird.  After the initial shock, I got to work defining just the styles I needed to make the site look the way I wanted.  This was more work, involved more trial and error, but left me with a result that is much leaner and meaner.&lt;/p&gt;

        <title>Unexplained CSV Whitespace</title>
        <description>&lt;p&gt;&lt;strong&gt;TLDR&lt;/strong&gt;: Blank lines in a &lt;code class=&quot;highlighter-rouge&quot;&gt;.csv.erb&lt;/code&gt; template render as whitespace in the final CSV output.&lt;/p&gt;

&lt;p&gt;Using the method I &lt;a href=&quot;https://jonathanpike.net/2016/09/Exporting-CSV-From-Rails&quot;&gt;described&lt;/a&gt; for exporting CSV, I created a new template and was testing it.  When I downloaded the CSV file and opened it, it appeared that the file was empty.  What was going on? I noticed that the scroll bar was rather long, so I started scrolling… and scrolling… and scrolling, down 1369 lines.  There’s the data. Then, I looked at the total lines in the file:  2738, which is double 1369.  It appears that, somewhere, I was inserting 1368 blank lines into the file prior to the data being written.&lt;/p&gt;

&lt;p&gt;I started debugging my &lt;code class=&quot;highlighter-rouge&quot;&gt;.csv.erb&lt;/code&gt; template to see where this whitespace was coming from.  First, I set a breakpoint in the template right before rendering the CSV.  Ruby’s CSV library wraps a string in a CSV object, and I could verify that the string had no whitespace.&lt;/p&gt;

&lt;p&gt;Next, I started removing elements from the CSV template until I found the culprit.  What I found was that an entire section of the CSV document was where the problem was happening. No matter what subsection of that section I removed, the issue would still persist.  If I removed the entire thing, the output would begin at line one.&lt;/p&gt;

&lt;p&gt;Finally, I decided to try something that I thought was crazy: I had defined a few variables in ERB tags earlier in the document, and I had inserted a blank line between each of these for readability.  What if I removed those?  The output began at line one. Bingo! Not so crazy after all!&lt;/p&gt;

&lt;p&gt;If you have more whitespace than you bargained for in CSV output and are using a &lt;code class=&quot;highlighter-rouge&quot;&gt;.csv.erb&lt;/code&gt; view template, check for blank lines. These could be the culprit.&lt;/p&gt;
</description>

        <title>Exporting CSV From Rails In Multiple Formats</title>
        <description>&lt;p&gt;Exporting data from Rails to be used in different applications can be a tricky topic.  What format do you need to export to?  What if one export needs some data and another export needs other data?  I had to solve exactly this problem with CSV.  Ruby has an excellent &lt;a href=&quot;http://ruby-doc.org/stdlib-2.3.0/libdoc/csv/rdoc/CSV.html&quot;&gt;CSV Library&lt;/a&gt; that is ripe for use with Rails.  From my research, there are 2 basic ways of exporting CSV data from a Rails application that I have come across so far:  &lt;a href=&quot;https://www.lockyy.com/posts/rails-4/exporting-csv-files-in-rails&quot;&gt;calling &lt;code class=&quot;highlighter-rouge&quot;&gt;to_csv&lt;/code&gt; on a model object in the Controller&lt;/a&gt; and &lt;a href=&quot;http://nithinbekal.com/posts/rails-csv-export/&quot;&gt;generating CSV directly in the view&lt;/a&gt;.&lt;/p&gt;

&lt;p&gt;Calling &lt;code class=&quot;highlighter-rouge&quot;&gt;to_csv&lt;/code&gt; is a great option if you only ever need the CSV data in one format.  My problem was more complex:  I needed to allow the user to choose what format the CSV data was exported in, and then provide it in that format.  To solve this problem, I chose to render the CSV in the view.  Here’s how I did it:&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;The Controller&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;Rails controller actions respond to HTML by default.  Luckly, setting up your controller to respond to CSV is really easy using &lt;a href=&quot;http://api.rubyonrails.org/classes/ActionController/MimeResponds.html#method-i-respond_to&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;ActionController::MimeResponds#respond_to&lt;/code&gt;&lt;/a&gt;, as follows:&lt;/p&gt;

&lt;div class=&quot;language-ruby highlighter-rouge&quot;&gt;&lt;pre class=&quot;highlight&quot;&gt;&lt;code&gt;&lt;span class=&quot;k&quot;&gt;def&lt;/span&gt; &lt;span class=&quot;nf&quot;&gt;index&lt;/span&gt;
	&lt;span class=&quot;c1&quot;&gt;# Standard model queries here...&lt;/span&gt;
	
	&lt;span class=&quot;vi&quot;&gt;@template&lt;/span&gt; &lt;span class=&quot;o&quot;&gt;=&lt;/span&gt; &lt;span class=&quot;n&quot;&gt;params&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;[&lt;/span&gt;&lt;span class=&quot;ss&quot;&gt;:template&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;]&lt;/span&gt;

When the user chooses what format they would like the CSV exported as, the download link provides a param that tells the controller what  CSV template to render.&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;Bonus Tip:&lt;/strong&gt; Optionally, you can also set the &lt;code class=&quot;highlighter-rouge&quot;&gt;Content-Disposition&lt;/code&gt; header to make the CSV file download automatically and set a file name, like so:&lt;/p&gt;

&lt;div class=&quot;language-ruby highlighter-rouge&quot;&gt;&lt;pre class=&quot;highlight&quot;&gt;&lt;code&gt;&lt;span class=&quot;nb&quot;&gt;format&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;.&lt;/span&gt;&lt;span class=&quot;nf&quot;&gt;csv&lt;/span&gt; &lt;span class=&quot;k&quot;&gt;do&lt;/span&gt; 
	&lt;span class=&quot;n&quot;&gt;headers&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;[&lt;/span&gt;&lt;span class=&quot;s1&quot;&gt;'Content-Disposition'&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;]&lt;/span&gt; &lt;span class=&quot;o&quot;&gt;=&lt;/span&gt; &lt;span class=&quot;s2&quot;&gt;&quot;attachment;filename=Your-File-Name-Here.csv&quot;&lt;/span&gt;

&lt;p&gt;&lt;strong&gt;The View&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;Here’s where Ruby’s CSV Library comes in handy.  Create as many CSV templates (in &lt;code class=&quot;highlighter-rouge&quot;&gt;csv.erb&lt;/code&gt; format, so you can embed Ruby directly in the template) as you need and save them wherever it makes sense.  They will each look something like this (obviously, with variation in the data displayed):&lt;/p&gt;

&lt;div class=&quot;language-ruby highlighter-rouge&quot;&gt;&lt;pre class=&quot;highlight&quot;&gt;&lt;code&gt;&lt;span class=&quot;o&quot;&gt;&amp;lt;&lt;/span&gt;&lt;span class=&quot;sx&quot;&gt;% export &lt;/span&gt;&lt;span class=&quot;o&quot;&gt;=&lt;/span&gt; &lt;span class=&quot;no&quot;&gt;CSV&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;.&lt;/span&gt;&lt;span class=&quot;nf&quot;&gt;generate&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;(&lt;/span&gt;&lt;span class=&quot;s2&quot;&gt;&quot; &quot;&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;,&lt;/span&gt; &lt;span class=&quot;p&quot;&gt;{&lt;/span&gt; &lt;span class=&quot;ss&quot;&gt;headers: &lt;/span&gt;&lt;span class=&quot;p&quot;&gt;[&lt;/span&gt;&lt;span class=&quot;s2&quot;&gt;&quot;Array&quot;&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;,&lt;/span&gt; &lt;span class=&quot;s2&quot;&gt;&quot;Of&quot;&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;,&lt;/span&gt; &lt;span class=&quot;s2&quot;&gt;&quot;Headers&quot;&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;],&lt;/span&gt; &lt;span class=&quot;ss&quot;&gt;write_headers: &lt;/span&gt;&lt;span class=&quot;kp&quot;&gt;true&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;,&lt;/span&gt; &lt;span class=&quot;ss&quot;&gt;encoding: &lt;/span&gt;&lt;span class=&quot;s2&quot;&gt;&quot;UTF-8}) do |csv| %&amp;gt; 
	&amp;lt;% csv.add_row data %&amp;gt; 
&amp;lt;% end %&amp;gt;
&amp;lt;%= export.lstrip.html_safe %&amp;gt; 
&lt;/span&gt;&lt;/code&gt;&lt;/pre&gt;
&lt;/div&gt;

&lt;p&gt;Let me explain what’s going on:&lt;/p&gt;

&lt;ol&gt;
  &lt;li&gt;The &lt;code class=&quot;highlighter-rouge&quot;&gt;export&lt;/code&gt; variable is set to &lt;a href=&quot;http://ruby-doc.org/stdlib-2.3.0/libdoc/csv/rdoc/CSV.html#method-c-generate&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;CSV.generate&lt;/code&gt;&lt;/a&gt;.  This is the method that will be generating the actual CSV string that the user has requested.&lt;/li&gt;
  &lt;li&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;CSV.generate&lt;/code&gt; takes 2 arguments: a string and a hash of options.  I’ve passed a blank string and provided &lt;code class=&quot;highlighter-rouge&quot;&gt;headers&lt;/code&gt;, &lt;code class=&quot;highlighter-rouge&quot;&gt;write_headers&lt;/code&gt;, and &lt;code class=&quot;highlighter-rouge&quot;&gt;encoding&lt;/code&gt; as options. &lt;code class=&quot;highlighter-rouge&quot;&gt;headers&lt;/code&gt; is the content of the top row of your CSV file, and can be either an array of strings or a string separated by commas.  &lt;code class=&quot;highlighter-rouge&quot;&gt;write_headers&lt;/code&gt; specifies that the headers should be added to the CSV output (optional – &lt;code class=&quot;highlighter-rouge&quot;&gt;false&lt;/code&gt; by default).  &lt;code class=&quot;highlighter-rouge&quot;&gt;encoding&lt;/code&gt; allows you to specify an encoding for the file (also optional – I’ve chosen UTF-8).&lt;/li&gt;
  &lt;li&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;CSV.generate&lt;/code&gt; takes a block, in which you pass in the rows you actually want to add to the output.  You can iterate over your data however you normally would in a view file in this block, and then pass the result to &lt;code class=&quot;highlighter-rouge&quot;&gt;csv.add_row&lt;/code&gt;.&lt;/li&gt;
  &lt;li&gt;I explicitly have not printed anything in the view file until the last line.  If the last line was left out, the CSV download would be a blank file.  This is optional (you can alternatively remove the &lt;code class=&quot;highlighter-rouge&quot;&gt;export&lt;/code&gt; variable and call &lt;code class=&quot;highlighter-rouge&quot;&gt;CSV.generate&lt;/code&gt; in &lt;code class=&quot;highlighter-rouge&quot;&gt;&amp;lt;%= %&amp;gt;&lt;/code&gt; tags).  I did it this way because:&lt;/li&gt;
&lt;/ol&gt;

&lt;ul&gt;
  &lt;li&gt;My own CSV exports required that no blank lines be present in the export.  I am calling &lt;code class=&quot;highlighter-rouge&quot;&gt;lstrip&lt;/code&gt; because I found that a &lt;code class=&quot;highlighter-rouge&quot;&gt;0A&lt;/code&gt; (&lt;code class=&quot;highlighter-rouge&quot;&gt;LF&lt;/code&gt;) character was being prepended to my CSV string, inexplicably.&lt;/li&gt;
  &lt;li&gt;I am calling &lt;code class=&quot;highlighter-rouge&quot;&gt;html_safe&lt;/code&gt; to avoid HTML escaping in the output.  This should only be done if your data is known safe.&lt;/li&gt;
&lt;/ul&gt;

&lt;p&gt;&lt;strong&gt;The Link&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;Now, when your user is given the option to select a CSV download, you can provide options that will provide the name of your CSV template to your controller.  I implemented this by using a &lt;a href=&quot;http://api.rubyonrails.org/classes/ActionView/Helpers/FormTagHelper.html#method-i-select_tag&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;select_tag&lt;/code&gt;&lt;/a&gt; together with &lt;a href=&quot;http://api.rubyonrails.org/classes/ActionView/Helpers/FormOptionsHelper.html#method-i-options_for_select&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;options_for_select&lt;/code&gt;&lt;/a&gt; that gathered the options from a Hash I provided.  You can use whatever method provides the correct param to your controller.&lt;/p&gt;

&lt;p&gt;In just a few lines of code, Ruby and Rails provide everything needed to create a very flexible CSV exporting system. Hope this helps with your CSV exporting needs!&lt;/p&gt;

In the end, I chose Firehose because:&lt;/p&gt;

It felt so good getting my development environment set up, generating a new Rails application, writing my first MVC code, typing &lt;code class=&quot;highlighter-rouge&quot;&gt;rails server&lt;/code&gt;, and seeing the results instantly in my browser. He went out of his way to be helpful to me, often going over the alloted 1 hour time period we had.  Jeff is a really great mentor and developer – he never ceased to amaze me with his deep knowledge of practically any question that I could throw at him.  I can trace many of my major successes to our sessions.  I’m sure that the other Firehose mentors are of equal caliber.&lt;/p&gt;

        <title>Bootcamp Student to Junior Developer</title>
        <description>&lt;p&gt;Going from feeling &lt;a href=&quot;https://jonathanpike.net/2016/03/Not-Smart-Enough&quot;&gt;not smart enough to be a programmer&lt;/a&gt; to being a &lt;a href=&quot;http://www.thefirehoseproject.com/&quot;&gt;programming bootcamp&lt;/a&gt; student was a big change.  Getting my development environment set up, starting the initial lessons, and becoming part of the Firehose Community all produced a sense of relief and joy.  I was finally on the path to becoming a developer!  Fast forward 15 weeks and I actually was a developer.  I had completed my assigned tasks on the group &lt;a href=&quot;http://endgame-chess.herokuapp.com&quot;&gt;chess application&lt;/a&gt; and felt very proud of the results. My next challenge was to face the great unknown: the job market for bootcamp graduates.&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;Resume&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;My first task was to polish up my resume and include my new programming knowledge and experience. I also decided to build a simple &lt;a href=&quot;http://jonathanpike.net/resume/&quot;&gt;resume page&lt;/a&gt; on this site to display it.  After all, I’m a web developer – making a page for my resume seemed like the obvious thing to do.  After completing my resume, I solicited feedback from my mentors.  That was a fantastic decision, and I encourage you to do the same.  Some of the advice that I got was:&lt;/p&gt;

&lt;ol&gt;
  &lt;li&gt;
    &lt;p&gt;&lt;strong&gt;Remember your audience&lt;/strong&gt;. A technical hiring manager is someone who is, at very least, familiar with things like Rails or a pull request, making explanations of such things unnecessary.  Don’t waste words (and space) on something that isn’t helping the hiring manager make a decision about whether you’re right for the job.&lt;/p&gt;
  &lt;/li&gt;
  &lt;li&gt;
    &lt;p&gt;&lt;strong&gt;Make it easy for the hiring manager&lt;/strong&gt;.  Your resume is your first point of entry into an organization, not the end-all-be-all of you getting hired.  Pay attention to what the job requirements are and check as many of those boxes as you have skills and experience.  Make it easy for the hiring manager to move your resume to the yes pile for the next round.&lt;/p&gt;
  &lt;/li&gt;
  &lt;li&gt;
    &lt;p&gt;&lt;strong&gt;Be honest&lt;/strong&gt;.  Don’t proclaim that you “know” something just because you’ve used it once, but don’t be afraid to say where your interests lie and what you’re currently learning.  Being a developer means you’re in for a career of constant learning and growing.  Show your curiosity.&lt;/p&gt;
  &lt;/li&gt;
  &lt;li&gt;
    &lt;p&gt;&lt;strong&gt;The more the merrier&lt;/strong&gt;. Another great decision I made was to not limit myself to a single resume location.  I mirrored the information in my resume to &lt;a href=&quot;https://ca.linkedin.com/in/pikejonathan&quot;&gt;LinkedIn&lt;/a&gt; and &lt;a href=&quot;http://stackoverflow.com/cv/jonathanpike&quot;&gt;Stack Overflow Careers&lt;/a&gt;, allowing more potential employers to stumble accross one of my profiles.  One benefit of the latter was that it also produced a nice PDF to submit via e-mail, for employers who wanted that.&lt;/p&gt;
  &lt;/li&gt;
&lt;/ol&gt;

&lt;p&gt;&lt;strong&gt;Job Hunting&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;Once my resume was ready, I took to looking for jobs.  My approach here was not very scientific: visit a few job boards daily that good companies advertised on, make a list of all the positions that sounded interesting, and apply to those positions on my list.  My favourite job boards were:&lt;/p&gt;

&lt;ol&gt;
  &lt;li&gt;&lt;a href=&quot;http://stackoverflow.com/jobs&quot;&gt;Stack Overflow Jobs&lt;/a&gt;.  This was my most used job board.  There are a lot of companies that post here every day and the filters are great.  If you create a profile, they will also match jobs with your interests and skills automatically.&lt;/li&gt;
  &lt;li&gt;&lt;a href=&quot;https://weworkremotely.com/&quot;&gt;We Work Remotely&lt;/a&gt;.  This board is &lt;em&gt;the&lt;/em&gt; board to look at for remote work.  There are fewer listings, but they’re all guaranteed to actually be hiring remotely.&lt;/li&gt;
  &lt;li&gt;&lt;a href=&quot;https://www.themuse.com/jobs&quot;&gt;The Muse Jobs&lt;/a&gt;.  The “Filter Jobs” feature on this board is great to look at larger companies in your area.  The profiles they have of the companies you’re looking at are the best feature here.&lt;/li&gt;
&lt;/ol&gt;

&lt;p&gt;Around the time I started looking for a job, &lt;a href=&quot;http://cobyism.com/remotejobseeker/&quot;&gt;The Complete Remote Jobseeker’s Handbook&lt;/a&gt;, an eBook from &lt;a href=&quot;https://github.com/cobyism&quot;&gt;Coby Chapple&lt;/a&gt; (a designer at GitHub), was fortuitously released.  This eBook is filled with great information for seeking remote work, including the benefits (and curses) of being a remote worker, lists of high quality job boards to look for remote work, strategies for applying for jobs (this was applicable to any job hunting, remote or not), and interviewing tips.  My biggest takeaway from the book was that the interviewer &lt;strong&gt;wants you to succeed&lt;/strong&gt;: finding and interviewing candidates is as much work for a company as applying and interviewing is for you.  If you’re the right person for the job, their work is over, as is yours.  It’s win-win!&lt;/p&gt;

&lt;p&gt;Another resource I found helpful was &lt;a href=&quot;https://www.interviewcake.com&quot;&gt;Interview Cake&lt;/a&gt;, a site that promises to teach you to be good at programming interviews.  While I didn’t actually face many technical challenges in the interviews I completed, this site was a great confidence booster as I realized that I was a better problem solver than I thought. And I learned a lot from the practice problems too!  Well worth the cost of admission.&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;Applying for Jobs&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;During my job hunt, I kept a detailed spreadsheet of the companies I applied for (including a copy of the job description, the date I applied, and the title of the job).  From late January to early April, I applied for &lt;strong&gt;23&lt;/strong&gt; jobs, which lead to &lt;strong&gt;4&lt;/strong&gt; interviews.  Early on, I spent a lot of time crafting a unique cover letter to each position I applied for, which meant that I was only applying to 2 positions a week (or less, depending on the time I had to spend on applications).  I asked &lt;a href=&quot;https://github.com/kenmazaika&quot;&gt;Ken Mazaika&lt;/a&gt; for some advice as I was going along, and he suggested I focus on a higher volume of applications, as (he estimated) that you have around a 10% chance of getting a response from companies for a junior role.  I took his advice and used the cover letters that I had already written to prepare a template.  This enabled me to apply to 5 or 6 a week (at peak).&lt;/p&gt;

&lt;p&gt;Of the interviews I had, &lt;strong&gt;2&lt;/strong&gt; were telephone screens and &lt;strong&gt;2&lt;/strong&gt; were in person interviews.  When it came to technical challenges, one of the telephone screens had me answering basic object oriented programming and responsive web design questions and one of the in person interviews had me filling out a general web development quiz.  I never had the dreaded whiteboard interview (whew!).  I was also given one take home assignment: building a simple &lt;a href=&quot;http://wc-todo.herokuapp.com&quot;&gt;to do app&lt;/a&gt; with the technology of my choice.&lt;/p&gt;

&lt;p&gt;A real surprise came when I was contacted on LinkedIn about a potential job opportunity – a company in Toronto was hiring Rails developers, and the recruiter noticed I had some experience with Ruby and Rails.  After a phone screen, 2 in person interviews, and a take home assignment, I was given a job offer from &lt;a href=&quot;http://www.bioconnect.com&quot;&gt;BioConnect&lt;/a&gt; to work on their Rails time and attendance application, &lt;a href=&quot;http://www.teamworks.cloud&quot;&gt;TeamWorks&lt;/a&gt;.  I accepted that offer and have been working on TeamWorks since the end of May.  My first month as a software developer has been better than I ever could have expected.  I work with a great team, I’ve been given the opportunity to learn a lot of new things, and I’m doing something that I really enjoy.&lt;/p&gt;

&lt;p&gt;If you’re currently in a bootcamp program, fear not about the great unknown of the job market.  You may face a lot of rejection (or be ignored) on the road to getting your first job, and you may feel out of your league at times. &lt;p&gt;&lt;strong&gt;The View&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;Rails expects a file in the views directory with the same name as the controller action and the same extension as the type of response.  To respond to the &lt;code class=&quot;highlighter-rouge&quot;&gt;create&lt;/code&gt; action with JavaScript, we need to make &lt;code class=&quot;highlighter-rouge&quot;&gt;create.js.erb&lt;/code&gt;:&lt;/p&gt;

&lt;div class=&quot;language-javascript highlighter-rouge&quot;&gt;&lt;pre class=&quot;highlight&quot;&gt;&lt;code&gt;&lt;span class=&quot;nx&quot;&gt;$&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;(&lt;/span&gt;&lt;span class=&quot;s1&quot;&gt;'.todos'&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;).&lt;/span&gt;&lt;span class=&quot;nx&quot;&gt;html&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;(&lt;/span&gt;&lt;span class=&quot;s2&quot;&gt;&quot;&amp;lt;%= j render partial: 'todos' %&amp;gt;&quot;&lt;/span&gt;&lt;span class=&quot;p&quot;&gt;)&lt;/span&gt;

&lt;p&gt;Let me explain what is happening:&lt;/p&gt;

&lt;ol&gt;
  &lt;li&gt;If the Ajax request is successful, the controller responds with this file, which will re-render the &lt;a href=&quot;https://github.com/jonathanpike/wc-todo/blob/master/app/views/todos/_todos.html.erb&quot;&gt;todos partial&lt;/a&gt;.  This is why I needed to have the &lt;code class=&quot;highlighter-rouge&quot;&gt;before_action&lt;/code&gt; in the controller load all of my todos from the database. The &lt;code class=&quot;highlighter-rouge&quot;&gt;j&lt;/code&gt; method is an alias for &lt;a href=&quot;http://api.rubyonrails.org/classes/ActionView/Helpers/JavaScriptHelper.html#method-i-j&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;escape_javascript&lt;/code&gt;&lt;/a&gt;, which performs &lt;a href=&quot;http://stackoverflow.com/a/1623813/5639584&quot;&gt;2 important functions when responding with JavaScript&lt;/a&gt;.&lt;/li&gt;
  &lt;li&gt;If the Ajax request doesn’t pass validation (and therefore doesn’t save to the database), I’ve set &lt;code class=&quot;highlighter-rouge&quot;&gt;flash[:alert]&lt;/code&gt; to have a message.  The flash block will loop through the flash hash and then put the message in a &lt;code class=&quot;highlighter-rouge&quot;&gt;p&lt;/code&gt; tag (inside of a &lt;code class=&quot;highlighter-rouge&quot;&gt;div&lt;/code&gt; with the ID &lt;code class=&quot;highlighter-rouge&quot;&gt;flash-messages&lt;/code&gt;).  You’ll see I’ve set it to &lt;code class=&quot;highlighter-rouge&quot;&gt;fadeOut&lt;/code&gt; after 3 seconds (because the page won’t get refreshed to get rid of it).&lt;/li&gt;
&lt;/ol&gt;

&lt;p&gt;&lt;em&gt;This is not the only way to write the response template&lt;/em&gt;.  For example, you could append the individual new To Do to the to &lt;code class=&quot;highlighter-rouge&quot;&gt;.todo-list&lt;/code&gt; &lt;code class=&quot;highlighter-rouge&quot;&gt;ul&lt;/code&gt; instead of re-rendering the partial.&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;Putting it All Together&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;Here is the chain of events that happens when a To Do is added:&lt;/p&gt;

&lt;ol&gt;
  &lt;li&gt;A user fills out the form and clicks the “Add New To Do” button.  This submits the form via Ajax (thanks &lt;code class=&quot;highlighter-rouge&quot;&gt;remote: true&lt;/code&gt;!).&lt;/li&gt;
  &lt;li&gt;The Todos Controller receives the request, and saves the To Do to the database (since it passes the validation).  It will also load all of the To Dos into the instance variable &lt;code class=&quot;highlighter-rouge&quot;&gt;@todos&lt;/code&gt; to be accessible in the view.  Then, it responds with the &lt;code class=&quot;highlighter-rouge&quot;&gt;create.js.erb&lt;/code&gt; view template.&lt;/li&gt;
  &lt;li&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;create.js.erb&lt;/code&gt; renders the &lt;code class=&quot;highlighter-rouge&quot;&gt;todos.html.erb&lt;/code&gt; partial, which now includes the newly added To Do.  You now see your new To Do added to the page.&lt;/li&gt;
&lt;/ol&gt;

&lt;p&gt;All of this happened without the page being refreshed.  Awesome!&lt;/p&gt;

And if I want to change the colour in the future, I simply change it in a single place.&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;jQuery-UJS and rails.js&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;jQuery-UJS is an “unobtrusive scripting adapter for jQuery” that provides the following features:&lt;/p&gt;

&lt;ul&gt;
  &lt;li&gt;force confirmation dialogs for various actions;&lt;/li&gt;
  &lt;li&gt;make non-GET requests from hyperlinks;&lt;/li&gt;
  &lt;li&gt;make forms or hyperlinks submit data asynchronously with Ajax;&lt;/li&gt;
  &lt;li&gt;have submit buttons become automatically disabled on form submit to prevent double-clicking.&lt;/li&gt;
&lt;/ul&gt;

&lt;p&gt;The file that actually does all this is called &lt;a href=&quot;https://github.com/rails/jquery-ujs/blob/master/src/rails.js&quot;&gt;rails.js&lt;/a&gt;.  I’ve included snippets of code below, but please check out rails.js for yourself to explore deeper.&lt;/p&gt;

I’m not sure why both are needed.&lt;/p&gt;

&lt;p&gt;You will also see some events with &lt;code class=&quot;highlighter-rouge&quot;&gt;.rails&lt;/code&gt; appended, like on this line defining what happens if the element is a form &lt;em&gt;(marked with arrows)&lt;/em&gt;:&lt;/p&gt;

HTTP verb) and &lt;code class=&quot;highlighter-rouge&quot;&gt;URL&lt;/code&gt; to submit to using &lt;a href=&quot;https://api.jquery.com/data/#data2&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;.data()&lt;/code&gt;&lt;/a&gt;, which reads the specified data attributes from the element.  If they aren’t present, it defaults to the &lt;code class=&quot;highlighter-rouge&quot;&gt;method&lt;/code&gt; and &lt;code class=&quot;highlighter-rouge&quot;&gt;action&lt;/code&gt; attributes on the element, respectively.&lt;/li&gt;
  &lt;li&gt;Next, it build up a &lt;code class=&quot;highlighter-rouge&quot;&gt;options&lt;/code&gt; object to pass over to the Ajax function with a variety of standard Ajax options that immediately trigger custom Events (for your own event handlers to deal with).&lt;/li&gt;
  &lt;li&gt;Finally, the options object is passed onto &lt;a href=&quot;#ajax&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;rails.ajax&lt;/code&gt;&lt;/a&gt; to actually perform the Ajax request.&lt;/li&gt;
&lt;/ol&gt;

&lt;p&gt;On form submit, this event handler is called. Here’s a rundown of what it does:&lt;/p&gt;

&lt;ol&gt;
  &lt;li&gt;It checks the &lt;code class=&quot;highlighter-rouge&quot;&gt;data-confirm&lt;/code&gt; attribute (with the &lt;code class=&quot;highlighter-rouge&quot;&gt;rails.allowAction&lt;/code&gt; function, which will return true if no function stops it) to see if the action needs to be confirmed prior to proceeding.&lt;/li&gt;
  &lt;li&gt;Next, it checks for a &lt;code class=&quot;highlighter-rouge&quot;&gt;novalidate&lt;/code&gt; attribute, which indicates that the form is not validated upon submit.  If &lt;code class=&quot;highlighter-rouge&quot;&gt;novalidate&lt;/code&gt; is not present, it will check for blank inputs (with the &lt;code class=&quot;highlighter-rouge&quot;&gt;rails.blankInputs&lt;/code&gt; function).  If there are blank inputs, it will stop submission of the form.&lt;/li&gt;
  &lt;li&gt;Then it checks if &lt;a href=&quot;#isRemote&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;isRemote&lt;/code&gt;&lt;/a&gt; is true. If so, it will check if there is a file input that has content in it.  This will allow you to implement a custom Ajax file upload method.&lt;/li&gt;
  &lt;li&gt;Finally, it uses &lt;a href=&quot;#handleRemote&quot;&gt;&lt;code class=&quot;highlighter-rouge&quot;&gt;handleRemote&lt;/code&gt;&lt;/a&gt; to deal with the form submission, and returns false to cancel regular submission.&lt;/li&gt;
&lt;/ol&gt;

&lt;p&gt;&lt;strong&gt;Conclusion&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;jQuery-UJS clearly does a lot in 534 lines.  But it’s not magic!  The next time you’re able to just write &lt;code class=&quot;highlighter-rouge&quot;&gt;remote: true&lt;/code&gt; to submit a form via Ajax, remember that rails.js is saving you a lot of time by being awesome.&lt;/p&gt;
